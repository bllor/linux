파일 시스템이란
운영체제가 파일을 저장하거나 삭제하는 등의 관리를 하기위해 사용하는 것
파일을 찾기 쉽게 저장하게 도와준다.
Linux의 파일체제 ext3,ext4
window의 파일체제 nfs, fat etc

/boot 부팅에 핵심적인 커널 이미지와 필요한 정보 파일을 담고 있는 디렉터리
/root 홈사용자 디렉터리 /root != /
/dev  장치파일(하드디스크, 키보드, 디바이스 등)파일을 저장하는 디렉터리
/etc  모든 구성파일이 존재한다. os나 애플리케이션이 작동하는데 필요한 시스템 환경이 저장되어 있는 ㄷ렉터리
/bin ->/usr/bin 사용자가 사용하는 명령어
/sbin ->/usr/bin 시스템 또는 파일시스템의 명령어
/opt  외부 응용프로그램을 설치 및 저장할 때 사용한다.
/proc 모든 실행 프로세스를 위해 사용되고 파일을 생성하는 디렉터리
/lib -> usr/lib 명령과 앱에 필요한 c언어 라이브러리 파일
/tmp 임시 파일을 위한 디렉터리
/home 유저를 위한 디렉터리
/var  시스템 로그를 위한 디렉터리
/run 계속 실행되고 있는 프로세스를 저장하고 있는 경로
/mnt 외부의 파일을 마운트 시키는 데 사용한다.
/media cd롬에 사용된다.

Cd(change directory) 디렉터리 변경
Pwd 현재 디렉터리의 주소를 출력해준다.
Ls(list) 현재 워킹 디렉터리의 파일 및 디렉터리를 출력해준다.

Cd / :부모 다이렉터리 가장 위에 있는 디렉터리로 이동한다.
Ls -l : 이 디렉터리 안에 있는 모든 파일 및 디렉터리를 보여준다.
Cd .. : 이전 폴더로 이동할 수 있음
Cd : root폴더로 이동? 사용자 명이 입력된 폴더로 이동한다.

첫번째 열은 파일의 유형을 나타낸다.

마지막 열은 파일의 이름을 나타낸다.
 
-----------------------------------------------------------
리눅스의 파일 유형
- : 일반적인 파일(텍스트 파일 같은 것)
D : 디렉터리
L : 원본 파일을 대신해서 원본 파일을 다른 파일이름으로 지정한 것 == 바로가기 
C : 특별한 파일이나 device 파일 키보드나 cpu등의 파일
S : 소켓(네트워크의 입출력을 담당하는 API이며, 두 컴퓨터 사이에서 데이터를 주고 받을 수 있다.)
P : 파이프 파일 (프로세스 간 통신을 위한 특수 파일)
B : block device 블럭 단위로 데이터를 읽고 쓸 수 있는 디바이스 (sda, cd-rom)

절대경로 및 상대경로

절대 경로는 /로 시작한다.

파일 및 디렉터리 찾기

1.find
Find . -name "fileName"을 할 경우 fileName의 절대위치를 알 수 있다. 
Find / -name "fileName" 전체 시스템(루드 디렉터리)에서 파일 검색

find[option] path [expression]으로 다양하게 사용할 수 있다.
[1]option
-P : 심볼릭 링크를 따라가지 않고 심볼릭 링크 자체 정보 사용
-L : 심볼릭 링크에 연결된 파일 정보 사용
-H : 심볼링 링크를 따라가지 않으나, command line argument를 처리할 땐 예외
-D : 디버그 메세지 출력
*심볼릭 링크 : window의 바로가기 같은 기능 == 특정 파일이나 디렉터리에 대한 참조를 포함하는 파일


[2]expression
-name : 지정된 문자열 패턴에 해당하는 파일 검색
-empty : 빈 디렉터리 또는 크기가 0인 파일 검색
-delete : 검색된 파일 또는 디렉터리 삭제 
-exec : 검색된 파일에 대해 지정된 명령 실행
-path : 지정된 문자열 패턴에 해당하는 경로에서 검색
-print : 검색 결과를 출력, 검색 항목은 newline으로 구분
-print0 : 검색 결과를 출력, 검색 항목은 null로 구분
-size : 파일 크기를 사용하여 파일 검색
-type : 지정된 파일 타입에 해당하는 파일 검색
  ㄴ b : block special
  ㄴ c : character special
  ㄴ d : directory
  ㄴ f : regular file
  ㄴ I : symbolic link
  ㄴ p : FIFO
  ㄴ s : socket
  Ex ) "dog"라는 폴더 검색할 경우 find . -name "dog" -type d
-mindepth: 검색을 시작할 하위 디렉터리 최소 깊이 지정 
-maxdepth: 검색할 하위 디렉터리의 최대 깊이 지정
-atime : 파일 접근 시각을 기준으로 파일 검색
-btime : 파일 내용 및 속성 변경 시각을 기준으로 파일 검색
-ctime : 파일의 데이터 수정시각을 기준으로 파일 검색

[3]표현식의 연산자
(expression) : 표현식의 우선순위 선정
!expression : expression 결과에 not 연산
Expression -a expression : and 연산
Expression -o expression : or 연산
